# Corpus rarity from term-frequency rank

**Goal:** Replace manual doc-frequency (`x/y manuals`) with a numeric rarity score based on where a term sits in the corpus frequency distribution.

**Formula:** `corpusRarity = freqRankIndex / (vocabSize - 1)` where index 0 is the most common corpus token. OOV → 1.0. Phrase combine: max rarity for 2-word, min for 3+.

**Why:** Doc frequency across 5 manuals is too coarse — `terminal` (4/5) still ranks too high. Relative rank pushes head-of-distribution generics (`position`, `terminal`, `system`) down while preserving rare-term boosts (`biodrill`, `slave cylinder`).
