# Fix Prisma Alpine OpenSSL Crash

**Goal:** Fix container startup crash on Cloud Run caused by missing OpenSSL libraries required by Prisma engine on Alpine.

**Context:** Deploy revision on Cloud Run crashes with `PrismaClientInitializationError: Unable to require('/app/backend/node_modules/.prisma/client/libquery_engine-linux-musl.so.node')` and mentions missing `libssl.so.1.1`. This happens because `node:22-alpine` doesn't include OpenSSL by default, which is needed by the Prisma query engine.

---

## Changes

1. Add `openssl` to `apk add` in the `Dockerfile` to provide OpenSSL 3 for the Prisma engine to detect and run successfully.
