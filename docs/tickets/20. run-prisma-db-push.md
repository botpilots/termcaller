# Run Prisma DB Push on Cloud Run

**Goal:** Ensure the SQLite database schema is initialized on container startup so tables like `User` exist before the application serves requests.

**Context:** The application is crashing with `P2021 The table 'main.User' does not exist in the current database` when attempting to log in. This happens because the Cloud Run volume mount is an empty directory (or contains a blank SQLite file) and we aren't running `prisma db push` or `prisma migrate deploy` during deployment to create the tables.

---

## Changes

1.  Update `Dockerfile` `CMD` to run `npx prisma db push --accept-data-loss` before starting the server (`npx tsx src/index.ts`).
    *   This ensures the database file on the mounted volume is initialized with the correct schema on every startup.
    *   Using `db push` is ideal for hackathons and SQLite where we don't have a complex migration history to manage.
