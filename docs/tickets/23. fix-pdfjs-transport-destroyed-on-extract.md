# Fix PDF.js "Transport destroyed" during Extract

**Goal:** Stop fatal SSE errors when running keyword extraction on multi-page PDFs.

**Root cause:** `scanPdfPages` processes pages with `PDF_PAGE_CONCURRENCY` (6), but pdf.js does not support concurrent `getPage()` / operator-list / text-content calls on a single document. Parallel workers corrupt the worker transport → `Error: Transport destroyed`.

**Root cause (update):** `scanPdfPages` used `return mapWithConcurrency(...)` inside `try/finally`. The `finally` block called `session.close()` immediately when the promise was returned (before page work finished), destroying the pdf.js transport mid-scan.

**Approach:**
1. Add a small async mutex utility (serialize pdf.js reads).
2. Await `mapWithConcurrency` inside `try` before `finally` closes the session.
