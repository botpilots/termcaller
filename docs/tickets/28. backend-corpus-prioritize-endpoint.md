# Backend corpus prioritize endpoint

**Goal:** Replace `GET /api/corpus/word-rank` (~200 KB full vocabulary) with a stateless batch lookup that returns corpus rarity per term.

**Why:** The frontend only needs rarity scores for keywords presently in the list. Corpus lookup belongs on the server; sorting by frequency/rarity/both stays client-side.

**API:**

```json
POST /api/corpus/prioritize
{ "items": [{ "id": "kw-1", "term": "retention clip" }] }
→ { "items": [{ "id": "kw-1", "corpusRarity": 0.82, "inCorpus": true }] }
```

`id` is echoed for O(1) merge when reordering the keyword list.
