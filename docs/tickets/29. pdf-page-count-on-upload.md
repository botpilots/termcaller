# PDF page count on upload

**Goal:** Persist and display the page count of a PDF as soon as it is uploaded, before extraction or validation runs.

**Why:** Users need immediate feedback on document size when starting a new project. Extraction progress already reports total pages, but that only appears after Extract is clicked.

---

## 1. Overall Goal & Scope

- Add optional `pageCount` on `Project`.
- On `POST /api/projects/:id/upload`, open the PDF with pdf.js, read `numPages`, store on the project, return in the JSON response.
- Show page count in the dashboard header for the selected project.

## 2. Implementation

- [x] Prisma: `Project.pageCount Int?`
- [x] `getPdfPageCount(pdfPath)` utility (open → read → destroy)
- [x] Upload route persists `pageCount`
- [x] Frontend: type + header badge after upload / project select

## 3. Test criteria

- [x] Uploading a valid PDF stores non-null `pageCount` in SQLite.
- [x] Upload response and project list include `pageCount`.
- [x] Header shows “N pages” for the selected project when known.
