# Full Document Processing & Semantic Analysis

## 1. Database Schema Updates
Update `backend/prisma/schema.prisma` to support the new data:
- **Illustration**: Add `pageNumber` (Int) and `figureNumber` (String?).
- **Callout**: Add `actualIdentifier` (String?) and a relation to `Concept` (`conceptId`).
- Run Prisma migration.

## 2. Gemini Prompt & Schema Update
Update `backend/src/services/geminiService.ts`:
- Add `figureNumber` to the `extractedConcepts` JSON schema.
- Instruct the LLM to extract the figure number if visible in the text/image, or return "N/A".

## 3. Backend: PDF Upload, Batch Processing & SSE
- **Upload Endpoint**: Implement `/api/projects/:id/upload` to receive the PDF, save it, and trigger background processing.
- **Batch Processing**: Implement a queue that processes pages in chunks (e.g., 3 at a time) using `pdfParser` and `geminiService` to avoid rate limits.
- **SSE Endpoint**: Implement `/api/projects/:id/stream` to push Server-Sent Events:
  - `progress`: `{ current: 5, total: 40 }`
  - `keyword_extracted`: Pushes new or updated keywords and concepts as they are saved to the DB.

## 4. Frontend: Sidebar & Progress Bar
Update `frontend/src/pages/Dashboard.tsx`:
- Connect to the SSE endpoint upon project selection/upload.
- Add a progress bar above "EXTRACTED KEYWORDS" based on SSE progress events.
- Aggregate incoming keywords in real-time. Sort A-Z.
- Display occurrence counts: e.g., `screw (3)`.

## 5. Frontend: Main Table View
- Create a table view for the selected project/keyword.
- Columns: `Page`, `Figure Number`, `Callout`, `Concept` (Functional Description).
- Populate this table with the data streamed from the backend.

## 6. Backend: Semantic Similarity Endpoint
- Implement `/api/keywords/:id/analyze-similarity`.
- Fetch all concepts for the keyword.
- If `vectorEmbedding` is missing, call Gemini's embedding API (`text-embedding-004`) to generate and save them.
- Calculate the "centroid" (average vector) of all concepts for this keyword.
- Calculate the cosine similarity of each concept to the centroid.
- Flag concepts with a similarity score below a threshold (e.g., `< 0.85`) as outliers.

## 7. Frontend: Similarity Visualization
- Add an "Analyse similarities" button above the table.
- When clicked, call the similarity endpoint and open a new tab/view.
- **Visualization**: 
  - Group items into "Core Meaning" (high similarity) and "Outliers" (low similarity).
  - Display a visual similarity score (e.g., a colored progress bar or badge: Green for >90%, Yellow for 80-90%, Red for <80%).
  - *(Optional Hackathon Flex)*: Render a simple 2D scatter plot using a library like `recharts` if time permits.
