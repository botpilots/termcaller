# Document preview with keyword navigation

**Goal:** When a keyword is selected, show a scrollable PDF page preview beside the occurrences table, with a minimap marking occurrence pages and auto-highlight via pdf.js text-layer locate.

**Priority:** HIGH — core editor UX for verifying extracted terminology in context.

**Status:** IN PROGRESS (UI sketch + page image / locate APIs)

---

## 1. Layout

```
┌──────────┬─────────────────────┬──────────────────────────┬──┐
│ Browse   │ Keyword header      │                          │▌│
│ panel    │ Tabs (Occurrences)  │  Scrollable page images  │ │ minimap
│          │ Occurrences table   │  + highlight overlays      │ │
└──────────┴─────────────────────┴──────────────────────────┴──┘
```

- **Select keyword** → minimap ticks all occurrence pages; preview scrolls to first hit; locate runs for first row.
- **Click table row** → scroll to that page; locate term then callout id on that page.
- **Find on page** (later) → Tesseract fallback when text-layer locate returns no boxes.

## 2. Backend

| Endpoint | Purpose |
|----------|---------|
| `GET /api/projects/:id/pages/:n/image` | Cached WebP render (`uploads/page-cache/{projectId}/`) |
| `GET /api/projects/:id/pages/:n/locate?term=&callout=` | pdf.js text search → normalized `[0,1]` bounding boxes |

Auth via `Authorization` header or `?token=` (for `<img src>`).

## 3. Frontend components

| Component | Role |
|-----------|------|
| `KeywordDocumentView` | Split layout: table + preview |
| `DocumentPreview` | Virtualized-ish page strip, highlight SVG overlays |
| `PageMinimap` | Occurrence ticks + scroll thumb; click to jump |
