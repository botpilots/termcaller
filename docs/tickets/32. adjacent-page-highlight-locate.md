# Adjacent-page highlight locate

**Goal:** When text-layer locate finds no match on an occurrence's page, search previous and next pages so highlights work for figures at page boundaries.

**Priority:** MEDIUM — follows document preview (ticket 31).

**Status:** DONE

---

## Problem

Occurrences are tied to an illustration `pageNumber`, but the keyword or callout label may only appear in the PDF text layer on page N−1 or N+1 (figure at top/bottom of page).

Current `locateOnPdfPage` searches only the requested page.

## Approach

1. **Backend:** Extend locate to try pages `[N−1, N, N+1]` (clamped to document bounds). Prefer matches on N; if none, return first hit on adjacent pages.
2. **API response:** Each box includes `pageNumber` where it was found; top-level `matchedPage` field.
3. **Frontend:** Store highlights keyed by actual match page; if `matchedPage !== occurrence page`, scroll preview to `matchedPage` and pulse there. Show subtle hint in status bar ("term on page N−1").

## Test criteria

- [x] Term on adjacent page only → highlight appears on correct preview page
- [x] Term on occurrence page → unchanged behavior (no adjacent search needed if hit on N)
- [x] Callout fallback also searches adjacent pages
- [x] Commit when complete
