# Occurrence form with page.figure sidebar

**Goal:** Replace the occurrences table with a sidebar + form layout: square page.figure toggles on the left, editable fields for the selected occurrence on the right.

**Priority:** HIGH — editor UX for reviewing and correcting extracted terminology per figure.

**Status:** IN PROGRESS

---

## Layout

```
┌────────┬──────────────────────────────┐
│ 25.1   │ Callout    [________]        │
│▌25.2   │ Definition [____________]  │
│ 26.1   │                              │
└────────┴──────────────────────────────┘
```

- **Left rail:** square buttons labelled `page.figure` (e.g. `25.1`); active item shows blue left border (matches analysis sidebar UX).
- **Right panel:** form fields for the selected occurrence (callout, definition; figure mode adds term + anomaly).
- **Keyword view:** selecting an occurrence scrolls the document preview and runs locate (unchanged behaviour).

## Scope

- `OccurrencesEditor` component (replaces `OccurrencesTable` in keyword + figure views)
- `formatPageFigureId` / `figureOccurrenceKey` helpers
- Local draft state for field edits (no persistence API yet)

## Test criteria

- Keyword occurrences show one square per figure; clicking updates the form and preview.
- Figure occurrences use the same layout with term + anomaly fields.
- Edits persist while switching between occurrences in the sidebar.
