# Occurrence edit fields and new-concept confirm modal

**Goal:** Expand the term editor form with a term name field and confirm flow; renaming the term prompts a modal before save.

**Status:** IN PROGRESS

---

## UX

- **Fields (keyword Edit tab):** Term, Callout, Definition
- **Confirm button** saves the occurrence draft via `PATCH /api/keywords/:id/occurrences`
- If **Term** differs from the loaded value → italic hint below Confirm (no modal); save creates a new keyword + concept
- `ConfirmPromptModal` kept in codebase for reuse elsewhere

## Scope

- `ConfirmPromptModal` reusable component (retained, not used in this flow)
- `occurrenceEditService` + keywords route for persistence
- `useOccurrenceEditorState` tracks original drafts for change detection
