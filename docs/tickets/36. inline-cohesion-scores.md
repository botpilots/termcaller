# Inline definition cohesion scores

**Goal:** Replace the Similarity tab with inline POOR / FAIR / CLOSE scores next to each definition, computed locally from stored embeddings.

**Why:** Editors need immediate feedback on outlier definitions without switching tabs or running a separate analysis.

---

## Scope

- Remove Edit / Similarity tabs from keyword view
- Backend embeds concepts on creation and refreshes all keyword concept embeddings on occurrence save
- PATCH returns updated concepts with `vectorEmbedding`
- Frontend computes centroid similarity locally; shows score beside Definition label and under page.figure rail buttons
