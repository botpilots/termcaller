# Auto-merge curation (Phase 1)

**Goal:** After keyword extraction, automatically merge duplicate concepts within each keyword group when definition embeddings are ≥95% similar (cosine similarity).

**Why:** Extraction often produces near-duplicate definitions for the same source term; Phase 1 consolidates them without manual curation.

**Priority:** HIGH — foundation for the curation workflow.

**Status:** IMPLEMENTED

---

## Scope

### Backend service (`conceptMergeService.ts`)

- `AUTO_MERGE_THRESHOLD = 0.95`
- `autoMergeKeywordConcepts(prisma, keywordId)` — cluster concepts by embedding similarity, pick canonical (closest to centroid; tie: longest definition), re-point callouts, disconnect/delete merged concepts
- `autoMergeProjectKeywords(prisma, projectId, userId)` — verify ownership, run per keyword with 2+ concepts, refresh embeddings after merges

### API

- `POST /api/projects/:id/curate/auto-merge` — authenticated manual trigger; returns merge summary JSON

### Extraction hook

- `processingService.ts` — after successful PDF processing, runs `autoMergeProjectKeywords` (try/catch; does not fail extraction)

### Tests

- `backend/tests/conceptAutoMerge.test.ts` — unit tests for clustering and canonical selection with mocked vectors (no API calls)
