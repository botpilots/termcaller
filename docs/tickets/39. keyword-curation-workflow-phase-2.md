# Keyword curation workflow (Phase 2)

**Goal:** One standard merged concept per keyword; divergent figures shown as terminology warnings when cohesion < CLOSE (90%); TBX exports canonical definition only; sidebar warning badges.

**Depends on:** Phase 1 auto-merge (ticket 37), ignore soft exclude (ticket 38)

**Status:** IMPLEMENTED

---

## UX

- Remove OccurrenceRail + editable OccurrenceForm for keyword mode
- Single definition card (read-only definition + page.figure provenance for CLOSE figures)
- Terminology warning cards below (one per divergent figure, informational only)
- Click warning → scroll + highlight in document preview
- Sidebar amber badge when keyword has definition warnings
- No definition editing, branch, or ignore actions in this phase

## Backend

- `keywordCurationService.ts` — canonical pick, `definitionWarnings` split
- `GET /api/keywords/:id/curation` — returns `concept`, `definitionWarnings`, `hasDefinitionWarnings`
- `GET /api/projects/:id/curation-summary` — keyword id → hasDefinitionWarnings
- TBX export: one canonical `termEntry` per keyword via `pickCanonicalConceptForKeyword`
- Branch/ignore endpoints retained but not wired in UI (deprecated for curation workflow)

## Refinement (warnings, not actions)

Phase 2 initially used "Official", "Outliers", Branch, and Ignore. Refinement:

- Renamed API: `canonicalConcept` → `concept`, `outliers` → `definitionWarnings`, `hasPendingReview` → `hasDefinitionWarnings`
- Removed `applyOutlierAutoIgnore` from extract/auto-merge pipeline
- Warnings are informational; no auto-exclusion from TBX
- Header shows figure count only; sidebar shows "Warning" not "Review"

## Thresholds

- Auto-merge: ≥95% pairwise
- Standard vs warning: ≥90% to centroid = CLOSE
