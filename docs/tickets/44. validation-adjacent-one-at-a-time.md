# Validation: LLM-picked adjacent page, one at a time

**Goal:** When re-checking unreferenced callouts, let the model choose previous vs next from the current page context and fetch only one adjacent page per follow-up (up to two rounds).

**Problem:** Validation eagerly fetched N-1 and N+1 in parallel and sent both images in one follow-up, adding cost and noise.

**Approach:** On unreferenced callouts, ask Gemini which single adjacent page to check; fetch and send only that image; optionally run a second round for the other direction if labels remain.
