# Keyword document-wide term search

**Goal:** When a keyword is selected, find all text-layer occurrences of the source term across the PDF, highlight them, and let the user cycle between matches. Remove callout-number fallback that wrongly highlights callout labels when the term is missing on the occurrence page.

**Priority:** HIGH — fixes broken scroll-to-term in document preview.

**Status:** IN PROGRESS

---

## Problem

Per-occurrence locate searches the figure page ±1 adjacent pages for the source term, then falls back to the callout id (e.g. `7`). The LLM associates terms with illustration pages where the label may only exist in the image layer, not pdf.js text — so the callout fallback highlights unrelated numbers.

## Approach

1. **Backend:** `locateAllTermMatchesInDocument(pdfPath, term)` — scan every page, return grouped occurrences with normalized boxes.
2. **API:** `GET /api/projects/:id/term-matches?term=…`
3. **Remove callout fallback** from `locateOnPdfPageWithAdjacent` (per-page locate stays term-only).
4. **Frontend:** On keyword select, fetch all matches once; show all highlights; prev/next cycles `activeMatchIndex` and scrolls to that page.
5. **Initial match:** Prefer first hit on LLM figure pages; else nearest page to first figure page.

## Test criteria

- [ ] "Stepper motor" never highlights callout `7` when term is absent on occurrence page
- [ ] All document text-layer hits are returned and highlightable
- [ ] Match prev/next scrolls and pulses the active occurrence
- [ ] Warning click scrolls to warning page; user can still cycle all matches
