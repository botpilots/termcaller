# Validation: exclude dimensions and units from callout checks

**Goal:** Stop validation from flagging dimension annotations and unit labels (e.g. `m2`, `mm`, `M4`) as unreferenced callouts.

**Problem:** Gemini treated printed measurement/unit text on illustrations as callout leader labels that lacked a text explanation.

**Approach:** Strengthen CALLOUT_RULES in the validation prompt with explicit examples of non-callout annotation text, and extend `isPlausibleCalloutLabel` to filter common dimension/unit patterns in post-processing.
