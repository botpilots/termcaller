# Fix Docker Build After Removing canvas

**Goal:** Fix Cloud Run Docker build/runtime for PDF processing after removing unused `canvas` dependency.

**Context:** Build failed compiling `canvas` on Alpine (no prebuild for musl, missing pkg-config/cairo). PDF pipeline now uses `pdfjs-dist` + `gm`/ImageMagick only.

---

## Changes

1. Add `imagemagick` and `ghostscript` to Dockerfile for `gm` PDF page rendering at runtime.
2. Keep build tools (`python3`, `make`, `g++`) for `better-sqlite3` native compile during `npm ci`.
