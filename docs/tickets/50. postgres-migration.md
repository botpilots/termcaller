# Migrate to PostgreSQL & GCS Durable PDFs

**Goal:** Move off GCSFuse/SQLite to Postgres, and store uploaded PDFs in Google Cloud Storage instead of the ephemeral container disk. This prevents silent extraction failures, removes 400 errors after container restarts, and stops SQLite locking under concurrent load.

## Completed
1. **Removed `better-sqlite3`**: Updated `backend/package.json` to remove the sqlite driver and typings.
2. **Prisma Postgres**: Updated `schema.prisma` to `provider = "postgresql"`.
3. **Dockerfile streamlined**: Replaced `db push` with `prisma migrate deploy`, and dropped the sqlite C++ build tools (`python3`, `make`, `g++`).
4. **GCS PDF Logic**: Created `gcsPdfStorage.ts` to push PDFs to `${GCS_BUCKET_NAME}` and pull them when required for extraction/validation.
5. **Project Routes Updated**: `POST /:id/upload` now writes directly to GCS and persists the GCS URI `gs://...` into `project.pdfPath`.
6. **Deployment Files Updated**: `deploy.yml` and `deploy.sh` now expect `GCS_BUCKET_NAME`, inject `DATABASE_URL` via Secret Manager, and no longer mount the GCSFuse volume.

## Next Steps for the User

You'll need a PostgreSQL database to deploy this correctly.

1. **Set up PostgreSQL**:
   - For Cloud Run, provision a **Google Cloud SQL (PostgreSQL)** instance (or use Neon/Supabase).
   - Get the Postgres connection string (e.g. `postgresql://user:pass@host:5432/termcaller`).
   - Create a Secret in Google Secret Manager called `DATABASE_URL` with this connection string.

2. **Run the Initial Migration**:
   - Because we switched database providers, you need to initialize the Postgres schema.
   - Set the `DATABASE_URL` environment variable locally on your machine.
   - Run `cd backend && npx prisma migrate dev --name init`.
   - This creates a new migration in `backend/prisma/migrations`. Commit these files!

3. **Deploy**:
   - Once the migration folder exists, push to `master`. The GitHub action will deploy to Cloud Run.
   - On startup, the container runs `npx prisma migrate deploy` to ensure your Cloud Postgres DB is up-to-date.

4. **Add Gemini API Key to Secret Manager**:
   - The deploy script expects a secret called `GEMINI_API_KEY`. Add it to Secret Manager to avoid warnings.
