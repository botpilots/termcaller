# Fix Dashboard Keyword Selection UI

**Goal:** The main dashboard table should show callout rows for the sidebar-selected keyword only, not a flattened summary of all keywords.

**Context/Tech Stack:** React (Vite), `frontend/src/pages/Dashboard.tsx`

---

## 1. Overall Goal & Scope

The sidebar should be an A–Z list of extracted keywords with occurrence counts. Clicking a keyword selects it. The main content area displays a table (Page, Figure, Callout, Concept) scoped to that keyword only.

## 2. Discussion: Challenges & Solutions

- **Current behavior:** Sidebar expands every keyword with concept cards; the main table flatMaps all keywords.
- **Fix:** Add `selectedKeywordId` state, render a clickable keyword list in the sidebar, and filter table rows to the selected keyword. Move the processing progress bar above the keyword list per the original plan.

## 3. PR Implementation Plan

- **PR 1: Keyword selection UI**
  - **Goal:** Sidebar selection drives the main table view.
  - **Scope:** `Dashboard.tsx` — selection state, sidebar list, filtered table, progress bar placement.
  - **Test Criteria:** Selecting a keyword in the sidebar updates the main table to show only that keyword's callouts.
