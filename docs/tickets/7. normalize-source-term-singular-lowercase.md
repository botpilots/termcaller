# Normalize Source Terms to Singular Lowercase

**Goal:** Extracted keywords should always be singular and lowercase, regardless of how they appear in the document.

**Context/Tech Stack:** Gemini prompt (`geminiService.ts`), backend normalization on save (`processingService.ts`).

---

## 1. Overall Goal & Scope

Technical manuals often use plural nouns ("Screws", "Brackets") and mixed casing. Source terms must be normalized to singular lowercase (e.g. `screw`, `bracket`) for consistent keyword grouping in the sidebar.

## 2. Discussion: Challenges & Solutions

- **LLM instruction:** Update the Gemini prompt and JSON schema to require singular, lowercase `sourceTerm` values.
- **Backend safety net:** Normalize terms before keyword lookup/creation so inconsistent LLM output still deduplicates correctly.

## 3. PR Implementation Plan

- **PR 1: Source term normalization**
  - **Goal:** Singular lowercase keywords everywhere.
  - **Scope:** Prompt update, `normalizeSourceTerm` utility, processing pipeline.
  - **Test Criteria:** "Screws", "screw", and "Screw" map to the same keyword.
