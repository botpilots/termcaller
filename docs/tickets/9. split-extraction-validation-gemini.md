# Split Gemini Extraction and Validation

**Goal:** Speed up page processing by simplifying the extraction prompt/schema and moving callout anomaly detection to a separate validation service and API endpoint.

**Problems:**
- Heavy pages (many callouts, e.g. page 17) overload Gemini and block chunk-based parallel processing.
- Nine instruction bullets and three validation mechanisms in one schema increase latency and output size.

**Approach:**
1. **Extraction (`geminiService`)** — term extraction only: `extractedConcepts` with `calloutIdentifiers`, `sourceTerm`, `functionalDescription`, `figureNumber`. Empty `sourceTerm` when name not found in text. Group duplicate callouts. Drop `actualIdentifiers`, `unreferencedCallouts`, `uncalledReferences`.
2. **Validation (`geminiValidationService`)** — separate Gemini call + `POST /api/validation/callouts` for unreferenced callouts, uncalled references, and label mismatches.
3. **Concurrency** — replace fixed 6-page chunks with a 6-slot worker pool so slow pages do not block unrelated pages.
4. **Timeout** — 60s per Gemini round-trip; timed-out pages are skipped and processing continues.
5. **Adjacent-page follow-up** — empty `sourceTerm` triggers prev/next page images (replaces old `unreferencedCallouts` trigger).
